Add your code to version control

Now let’s add the code to the repository. In order to do that, right click on the “root” of your workspace and select “add directory tree to source control” as the following image shows:

Figure 1: Add directory

All the tree of code will be recursively added and your “items view” will look like this:

Figure 2: Tree View

As you can see everything is in “checkedout” status, which means Plastic SCM knows about all the “added files and directories” but they are not yet checked in to the repository.

In order to “checkin” (send everything to the repository), just right click on the “root of the workspace” and click on “checkin” as the image below shows:

Figure 3:

After clicking “checkin” a dialog will show up and will let you introduce a comment for your checkin (in my case I just typed “initial checkin”) and a list of the files to be added:

Figure 4: List the files to be added

Click on the “checkin” button to confirm the operation. Once the checkin completes, the “items view” will look like the following:

Figure 5: Status Item view

Please note the status of all files is now “controlled” except for the “ignored” ones we configured before

It means you have just put your first code inside Plastic SCM control! Congratulations.
