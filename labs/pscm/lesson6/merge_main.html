Merge back to “main”

Let’s go to “Branch Explorer” view to see the current status of the repository. Something like the following will show up:

Figure 1: Branch Explorer

Let’s now make a merge from SCM002 branch to main branch. Right click on SCM002 branch and select “Merge from this branch” option:

Figure 2: Merge from this branch

Once the button is clicked you will see how merging from SCM002 branch to main branch in “Merge from branch” view.

Figure 3: Merge Branch View

As you can see, there are two columns marked:

  • Remarks: short description of the conflict for each item. In this case, changes in both source and destination contributors.
  • Contributor: prints the actions that will be run when the “Process all merges” button is clicked.

Let’s go to know more about the merge. So, click “Explain Merge” button at the top in tool bar, “The Merge Explanation” view will look like the following:

Figure 4: Merge Explanation

On the figure you can learn how merge works. There are 3 important terms:

  • Source (src): it shows the revision you are merging from, SCM002 in this case.
  • Base: the initial revision, it’s Baseline002.
  • Destination (dst): what you have right now on your workspace. A new changeset will be created on main branch. It will contain the merge result.

Let’s return to “Merge form branch” view. Then click on the “Process all merges” button to start the process. Once the merging completes, the “Branch explorer” view will look like the following:

Figure 5: Branch Explorer

There is a new changeset pending to checkin. Let’s go to “Pending changes” view and check what we have modified so far:

Figure 6: Pending Changes

As you can see “the Pending Changes” view lists the file you previously merged. If you clicked on the “Show Diffs” button you will see the lines you’ve modified:

Figure 7: Pending Changes links

You can inspect the difference prior to checkin the merge operation. The code has been correctly merged.
