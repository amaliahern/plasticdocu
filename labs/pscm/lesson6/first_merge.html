Click on "Process all merges” button at the top of tool bar. The merge will be processed as the image below shows:

Figure 5: Process all merges

So, we did our first merge. There isn’t merge conflicts, now the main branch contains the changes made on the SCM001 branch.

Let’s go to “Branch Explorer” view to see the current status of the repository. Something like the following will show up:

Figure 6: Branch Explorer after merging

In the figure above there are two new things:

  • Checked out changeset: the “house” in dashed circle
  • Pending merge link: the dashed line with an ended arrow

So, what does it mean? We did a merge from SCM001 branch to main branch, so a new changeset is going to be created on checkin. It will contain the changes in the two files modified: ‘dokan-net-0.6.0\readme-dokan.txt’ and ‘dokan-net-0.6.0\license.txt’.

Let’s go to “Pending changes” view and you will see something like the following will show up:

Figure 7: Pending Changes

As you can see in the figure:

  • The branch, where you are going to checkin, is main branch.
  • The status si replaced on both files.

You can now checkin and don’t forget to write a comment like: “Merge from SCM001 branch to main branch”
